## Spacing Scale

Base unit: 8px. All values are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, color strip width, dot indicators |
| sm | 8px | Intra-card element spacing, WeekStrip day cell vertical padding |
| md | 16px | Card horizontal padding, section horizontal padding |
| lg | 24px | Section vertical padding, list outer padding |
| xl | 32px | ŌĆö (not used in this phase) |
| 2xl | 48px | ŌĆö (not used in this phase) |
| 3xl | 64px | ŌĆö (not used in this phase) |

Exceptions:
- Color strip on card left edge: 3px wide (design system "timeline segments" token)
- Card corner radius: 10px (design system "cards/goal" token)
- Week navigation bar vertical padding: 8px top/bottom, 16px horizontal (matches CalendarTabView pattern)
- WeekStrip day cell corner radius: 8px (matches WeekStripView existing implementation)
- Card vertical padding: 10px top/bottom (derived from 8px base with 2px accent for density)
- Gap between session cards: 8px

Source: `.design-engineer/system.md`, `CalendarTabView.swift`, `WeekStripView.swift`.

## Component Inventory

### SessionsTabView (new)

Top-level container view. Mirrors `CalendarTabView` structure without the timeline.

```
VStack(spacing: 0)
  WeekNavigationBar          ŌåÉ existing pattern from CalendarTabView
  WeekStripView              ŌåÉ reuse as-is, zero changes
  ScrollView(.vertical)
    VStack(spacing: 8)
      ForEach(sessions)
        SessionCardView      ŌåÉ new
```

### WeekNavigationBar (inline in SessionsTabView)

Reuse the exact HStack pattern from `CalendarTabView` body:
- Left: `chevron.left` plain button ŌĆö shifts week back
- Center: "Today" plain button in `CategoryColors.accent`
- Right: `chevron.right` plain button ŌĆö shifts week forward
- No calendar filter menu (that is Calendar-tab-specific)
- Horizontal padding: 16px, vertical padding: 8px

### WeekStripView (reused, unchanged)

Zero modifications. Pass `selectedDate`, `dailyTotals`, `onSelectDate` as already defined.

### SessionCardView (new)

Card anatomy (left to right, top to bottom):

```
HStack(spacing: 0)
  Rectangle (color strip)    3px wide, full card height, category color, cornerRadius on leading edge only
  VStack(spacing: 4)         horizontal padding: 12px left, 12px right; vertical padding: 10px
    HStack
      Text(category)         13px semibold, Theme.textPrimary
      Spacer
      Text(duration)         12px semibold, Theme.textSecondary  e.g. "45m"
    Text(intention)          13px regular, Theme.textSecondary (if nil: "No intention" in Theme.textQuaternary)
    Text(timeRange)          10px regular, Theme.textTertiary  e.g. "09:15 ŌĆō 10:00"
    if expanded:
      Divider                Theme.border, top margin 6px
      AppUsageListView
```

Card container:
- Background: `Theme.backgroundSecondary`
- Corner radius: 10px
- Border: 0.5pt stroke `Theme.border`
- No shadow (depth strategy: borders only per design system)
- `.contentShape(Rectangle())` for full hit area
- Tap gesture toggles `expandedSessionId` (accordion: sets to this id, or nil if already expanded)

Live session indicator:
- Today's in-progress (currentSession): add a 4px terracotta dot left of the category name, same row

### AppUsageListView (new, shown inside expanded card)

Visual treatment: simple vertical list. No mini bars (insufficient horizontal space in dense card). Chosen by Claude's discretion per CONTEXT.md D-05 and design system density principle ("a watch face you glance at").

```
VStack(spacing: 4)
  ForEach(appsUsed)
    HStack
      Text(appName)         10px regular, Theme.textSecondary
      Spacer
      Text(duration)        10px semibold, Theme.textTertiary  e.g. "45m"
```

Padding: 0px ŌĆö inherits card's 12px horizontal padding. Top padding: 6px above first row (after divider).

### EmptyStateView (new, inline in SessionsTabView)

```
VStack(spacing: 8)
  Text("No sessions")        13px regular, Theme.textTertiary
```

Centered in the scroll area. No illustration, no action button ŌĆö matches D-06.

## Interaction Contract

### Accordion Expansion (DETAIL-01, DETAIL-02, DETAIL-03)

State: `@State private var expandedSessionId: UUID?`

- Tapping a collapsed card: set `expandedSessionId = session.id`
- Tapping an already-expanded card: set `expandedSessionId = nil`
- Changing `selectedDate`: set `expandedSessionId = nil` (explicit `onChange`)
- Animation: `.animation(.easeInOut(duration: 0.2), value: expandedSessionId)` on the card's content height

Only one card expanded at a time. This is enforced by the single `UUID?` state ŌĆö not a Set.

### Week Navigation (NAV-02)

- Chevron buttons: call `shiftWeek(-1)` / `shiftWeek(1)` ŌĆö same pattern as CalendarTabView
- "Today" button: set `selectedDate = Date()`, reload week sessions
- Week boundary: no hard limit ŌĆö allows browsing historical weeks indefinitely

### Day Selection (NAV-03)

- Tap any day cell in WeekStrip: `selectedDate = day`, triggers `loadWeekSessions()` via `.onChange`
- Selected day cell: accent tint background (`CategoryColors.accent.opacity(0.1)`), accent text
- Today dot: 4px filled circle, `CategoryColors.accent`, shown only when day is today

### Live Today Merge (LIST-03)

Computed property `selectedDaySessions` ŌĆö identical pattern to `CalendarTabView.selectedDaySessions`:
- CloudKit sessions for the day, deduplicated against live IDs
- Append `sessionEngine.todaySessions`
- Append `sessionEngine.currentSession` if non-nil
- Sort by `startTime` ascending

### Tab Registration (NAV-01)

New `AppTab.sessions` case:
- Raw value: `"Sessions"`
- Icon: `"list.bullet.rectangle"` (SF Symbols ŌĆö represents a structured list, distinct from calendar and clock)
- Position: between `.today` and `.calendar` in `AppTab.allCases` order

Tab bar renders `Sessions` between Today and Calendar.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Tab label | "Sessions" |
| Week navigation ŌĆö today button | "Today" |
| Empty state | "No sessions" |
| No intention placeholder | "No intention" |
| In-progress session label | (no text label ŌĆö indicated by terracotta dot only) |
| Error state (load failure) | "Couldn't load sessions" |
| Section header (none declared) | No section headers in this phase |

Notes:
- "No sessions" is the complete empty state ŌĆö no subtext, no call to action (D-06: minimal, no illustrations or hints)
- "No intention" renders in `Theme.textQuaternary` (most muted text token) to signal it is a placeholder, not content
- Duration formatting: use shortest form ŌĆö "45m" for under 1 hour, "1h 12m" for 1+ hours, "2h" if exactly on the hour
